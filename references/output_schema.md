# 输出数据结构与文档结构

## `<标题>_document.json`（Step 3.5/3.6/3.7 写入，Step 3.8 直接生成 .docx）

```json
{
  "title": "<拍摄时间+人物简介，如 26-0509 车辆学院直博生>",
  "frame_path": "人物静帧.jpg 或 null（音频输入时为 null）",
  "input_type": "video 或 audio",
  "source_file": "<原始输入文件名，如 输入.mp4 / 输入.m4a>",
  "transcription_tool": "通义千问 Qwen3-ASR-Flash（阿里云百炼）或 SenseVoice/Paraformer 模型名",
  "speaker_method": "CAM++ 说话人嵌入（FunASR spk_model，本地）或 LLM 语义切分（云端 qwen-plus，免 HF Token）",
  "summary_method": "LLM 生成",
  "date": "YYYY-MM-DD",
  "summary": "<LLM 生成的 3-5 句话总结性摘要，多段落用 \\n 分隔>",
  "summary_sections": [
    {
      "title": "<板块标题，如『板块一：受访人背景』『板块二：核心观点』>",
      "content": "<该板块的摘要内容，多段落用 \\n 分隔>"
    }
  ],
  "person_info": [
    {
      "name": "人物（张同学）",   // 表格标题；单人可写"人物"或留空
      "fields": [
        {"field": "学校/单位", "value": "..."},
        {"field": "专业/学院", "value": "..."},
        {"field": "年级/身份", "value": "..."},
        {"field": "家乡", "value": "..."},
        {"field": "关键经历", "value": "..."},
        {"field": "核心观点", "value": "..."}
      ]
    }
    // 若有多位人物，再追加一个对象（渲染为独立表格）
  ],
  "conversation": [
    {"speaker": "说话人1", "timestamp": "[00:00]", "text": "<对话内容>"},
    {"speaker": "说话人2", "timestamp": "[00:15]", "text": "<对话内容>"},
    {"speaker": "说话人1", "timestamp": "[01:30]", "text": "<对话内容>"}
  ]
}
```

- `frame_path` 为 `null`（音频输入）时，`build_docx.py` 自动跳过静帧插入。
- `summary` 缺失时「内容摘要」章节自动省略（但若有 `summary_sections`，仍会渲染分板块）。
- `summary_sections` 为空数组 `[]` 或缺失时，仅显示 `summary` 一句总结性摘要（无板块）。
- `person_info` 为空数组 `[]` 或缺失时，**整段「人物信息」板块自动省略**（未透露任何个人信息时不展示空白表格）。
- `person_info` 含多个对象时，**每位人物渲染为独立表格**（标题取各自 `name`）；单人则只渲染一个表格。

## 最终 `.docx` 文档结构（build_docx.py 渲染结果）

```
# <标题>

<居中人物静帧，仅视频输入；音频输入时无此图>

📝 内容摘要
<LLM 生成的 3-5 句话总结性摘要>

### 板块一：<板块标题>      ← H2 子标题，从 summary_sections[].title 渲染
<该板块的摘要内容>

### 板块二：<板块标题>
<该板块的摘要内容>

👤 人物信息（无信息则该板块整段省略；多人时每位人物一个独立表格）

— 人物（张同学）—
| 字段 | 内容 |
|------|------|
| 学校/单位 | ... |
| ...  | ... |

— 人物（李同学）—        ← 第二位人物：再多用一个表格
| 字段 | 内容 |
|------|------|
| 学校/单位 | ... |
| ...  | ... |

📋 文档信息
> 源文件：<source_file>
> 输入类型：视频 / 音频
> 转录工具：Qwen3-ASR-Flash / SenseVoice / Paraformer
> 说话人识别：CAM++ 说话人嵌入（本地）/ LLM 语义切分（云端）
> 转录日期：YYYY-MM-DD

💬 对话记录
🔴 **说话人1** [00:00]    ← 说话人1 用 🔴红色圆点
<对话内容>

🔵 **说话人2** [00:15]    ← 说话人2 用 🔵蓝色圆点（与说话人1 的红色色相差异大，肉眼一目了然）
<对话内容>

🟢 **说话人3** [00:30]    ← 说话人3 用 🟢绿色圆点
<对话内容>

🟡 **说话人4** [00:50]    ← 说话人4 用 🟡黄色圆点
<对话内容>
```

> 💡 说话人区分使用**色相差异大的彩色圆点** emoji（🔴红 → 🔵蓝 → 🟢绿 → 🟡黄 → 🟣紫 → ⚫黑 → 🟤棕 → ⚪白 → 🟠橙 → 🔘灰 循环）。配色按色相分散排列，相邻说话人拿到的颜色色相差距大（默认说话人1=红、说话人2=蓝），避免色相近的颜色（如红+橙、蓝+绿）相邻。同一说话人跨多轮始终用同一颜色，便于肉眼一眼分清。
> 📝 内容摘要板块同时包含：① 总结性摘要（一段话）② 分板块摘要（H2 板块小标题），板块顺序由 LLM 决定（如「受访人背景 / 核心观点 / 关键经历」等）。
